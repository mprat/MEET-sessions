You are almost at the end of your MEET journey. It's time to do an
individual project.\subsection{Mobile Technology / Hardware}\begin{itemize}
\item
  How many students in the room have a mobile device?
\end{itemize}\begin{itemize}
\item
  Mobile is a VENUE, a SPACE, a DEPLOY MECHANISM
\item
  Hardware is a VENUE, a SPACE, a DEPLOY MECHANISM
\end{itemize}\subsection{Unique features of mobile (reaching users)}

Things you can't get with web or desktop apps\begin{itemize}
\item
  With the user at all times (small)
\item
  Location information
\item
  Take photos
\item
  Can detect when you shake it (accelerometer)
\end{itemize}\subsection{Unique features of hardware (reaching users)}

Things you can't get with web or desktop or mobile apps\begin{itemize}
\item
  Physical input from user (button, dial)
\item
  Measurement from physical world (sensor)
\item
  Control something in physical world (robot)
\end{itemize}\subsection{You and Technology Tools}

There are a million tools out there for development. You can't be an
expert in one and that's it. You constantly need to learn more.

\begin{itemize}
\item
  Tutorials are THE way to learn things
\item
  Want to get used to mobile space but still practice learning from
  tutorials, so we are choosing an ``easier'' technical tool
\item
  Same for hardware space
\end{itemize}\subsection{Project tools}

\begin{itemize}
\item
  Design (both how it looks and how it works) is as important than being
  able to use the tools
\item
  We will practice having a plan and doing it

\begin{itemize}
\item
  Prototyping tools
\item
  Impact, prevalence of Android. How many students in the room have
  Android?
\end{itemize}\subsection{Briefly about Android apps}

\begin{itemize}
\item
  Android phones only understand Java
\item
  We will not use Java, but will use something that gets translated
  (compiled) into Java
\item
  App gets put into a package (a \textbf{.apk} file) that is all Java
\item
  Android phones can install arbitrary \textbf{.apk} files if you ask
  nicely
\end{itemize}\subsection{Terms}

\begin{itemize}
\item
  \textbf{accelerometer} - measures phone movement
\item
  \textbf{apk} - file extension for the bundle of files that make an app
\item
  \textbf{application / app} - bundle of files (compiled into Java) that
  have a specific purpose
\item
  \textbf{camera} - can be front or back facing
\item
  \textbf{event handler} - what happens as a result of something else
  happening. Listens for something to happen (the \emph{event}) and
  triggers something else
\item
  \textbf{screen / touchscreen}
\end{itemize}\subsection{Our tool: AppInventor}
